# Framework Template

This template provides the structure for all framework files in the Strategic Frameworks library. Follow this format exactly when creating new framework files.

---

## File Naming

**Format:** `[number]-[framework-name].md`

**Rules:**
- Number: Sequential (next available number)
- Framework name: kebab-case (lowercase with hyphens)
- Example: `39-scamper-technique.md`, `40-balanced-scorecard.md`

## Formatting Guidelines

### Headers
- Use ATX-style headers (`#`, `##`, `###`)
- Framework name is `#` (H1)
- Major sections are `##` (H2)
- Subsections are `###` (H3)

### Lists
- Use `-` for unordered lists
- Use `1.` for ordered lists when sequence matters
- Indent nested lists with 2 spaces

### Bold and Emphasis
- **Bold** for important terms, names, labels
- *Italic* for book titles, emphasis (use sparingly)

### Code Blocks
- Use for examples of questions or templates if helpful
- Not typically needed for framework files

### Links
- Full URLs in brackets: `[Link Text](https://url.com)`
- Citations with full details in Sources section

---

## Content Guidelines

### Voice and Tone
- **Professional but accessible** - Write for practitioners
- **Specific and concrete** - Real examples, not hypotheticals
- **Action-oriented** - Focus on what users do with framework
- **Honest about limitations** - Note when framework doesn't fit

### Quality Standards
- **Accuracy** - All facts verified, sources cited
- **Completeness** - All template sections filled thoroughly
- **Clarity** - Anyone can understand without prior knowledge
- **Practicality** - Users can apply framework after reading

### Common Mistakes to Avoid
❌ Vague examples ("A company used this to improve their process")
❌ Missing source citations
❌ Academic jargon without explanation
❌ Incomplete sections
❌ Generic descriptions that could apply to any framework
❌ Made-up or AI-generated content

✅ Specific examples with names and details
✅ Proper citations for all claims
✅ Clear language accessible to practitioners
✅ All sections thoroughly completed
✅ Unique characteristics clearly explained
✅ Real, documented frameworks with proven track records
