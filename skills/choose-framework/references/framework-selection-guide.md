# Framework Selection Guide

This guide explains the 6 key dimensions used to match frameworks to situations. Use this to understand what to explore in each dimension and how to match frameworks effectively.

---

## Dimension 1: Problem Type

### What to Explore

The fundamental nature of what the user is trying to accomplish.

**Key questions:**
- Is this about choosing direction (strategic)?
- Is this about improving execution (operational)?
- Is this about creating something new (innovation)?
- Is this about making a choice (decision)?

### Problem Type Categories

#### Strategic Problems
**Characteristics:**
- Long-term direction
- Competitive positioning
- Market dynamics
- Organizational alignment

**Best frameworks:**
- Porter's Five Forces (competitive analysis)
- Blue Ocean Strategy (uncontested markets)
- Wardley Mapping (strategic evolution)
- Scenario Planning (uncertain futures)
- McKinsey 7S (organizational alignment)
- BCG Matrix (portfolio management)
- Ansoff Matrix (growth strategies)

**Why it matters:** Strategic frameworks focus on analysis and positioning, not immediate action.

#### Operational Problems
**Characteristics:**
- Process improvement
- Execution challenges
- Resource allocation
- System optimization

**Best frameworks:**
- Theory of Constraints (bottleneck identification)
- Pareto Principle (80/20 focus)
- OKR Framework (execution alignment)
- Systems Thinking (interconnections)
- OODA Loop (fast execution)

**Why it matters:** Operational frameworks focus on doing things better, not choosing what to do.

#### Innovation Problems
**Characteristics:**
- Creating new products/services
- Solving unmet needs
- Experimentation required
- User-centered design

**Best frameworks:**
- Design Thinking (human-centered innovation)
- Jobs-to-be-Done (understanding needs)
- Lean Startup (validated learning)
- First Principles (rebuilding from fundamentals)
- Blue Ocean Strategy (market creation)
- Six Thinking Hats (creative exploration)

**Why it matters:** Innovation frameworks focus on discovery and iteration, not optimization.

#### Decision Problems
**Characteristics:**
- Choosing between options
- Evaluating trade-offs
- Managing risk and uncertainty
- Need for clarity

**Best frameworks:**
- Mental Models (Munger's multidisciplinary thinking)
- Regret Minimization (long-term perspective)
- Pre-Mortem Analysis (imagining failure)
- Second-Order Thinking (consequences of consequences)
- Cynefin Framework (understanding context)
- Inversion (thinking backwards)
- Eisenhower Matrix (prioritization)

**Why it matters:** Decision frameworks focus on choice-making process, not analysis.

### Common Mistakes

❌ **Treating operational problems as strategic** (using Porter's Five Forces to optimize a process)
❌ **Treating strategic problems as operational** (using Pareto Principle for market positioning)
❌ **Confusing innovation with optimization** (using Lean Startup to improve existing product)
❌ **Using decision frameworks for analysis** (using Regret Minimization instead of competitive analysis)

## Dimension 2: Situation Context

### What to Explore

The nature of the environment and how much is known/stable.

**Key questions:**
- How clear is cause and effect?
- How much is changing?
- How predictable is the environment?
- What's the level of complexity?

### Context Categories (Cynefin-inspired)

#### Stable/Clear Context
**Characteristics:**
- Clear cause-effect relationships
- Best practices exist
- Predictable outcomes
- Low uncertainty

**Best frameworks:**
- SWOT Analysis (structured analysis)
- Porter's Five Forces (industry structure)
- Eisenhower Matrix (clear prioritization)
- Pareto Principle (known patterns)
- Business Model Canvas (established models)

**Why it matters:** In stable contexts, analytical frameworks that leverage existing knowledge work best.

#### Changing/Competitive Context
**Characteristics:**
- Evolving situation
- Competitive dynamics
- Need for adaptability
- Medium uncertainty

**Best frameworks:**
- OODA Loop (fast adaptation)
- Wardley Mapping (evolution tracking)
- Blue Ocean Strategy (changing game)
- Scenario Planning (multiple futures)
- Lean Startup (rapid iteration)

**Why it matters:** In changing contexts, adaptive frameworks that enable quick response work best.

#### Uncertain/Unpredictable Context
**Characteristics:**
- Many unknowns
- Unclear cause-effect
- High risk
- Need for experimentation

**Best frameworks:**
- Lean Startup (validated learning)
- Pre-Mortem Analysis (anticipating failure)
- Scenario Planning (multiple paths)
- Inversion Thinking (avoiding failure)
- Risk Management Frameworks

**Why it matters:** In uncertain contexts, frameworks that embrace unknowns and enable learning work best.

#### Complex/Emergent Context
**Characteristics:**
- Interconnected systems
- Emergent patterns
- Non-linear relationships
- Requires sense-making

**Best frameworks:**
- Cynefin Framework (understanding complexity)
- Systems Thinking (interconnections)
- Mental Models (multidisciplinary)
- First Principles (fundamental truths)
- OODA Loop (rapid sensing)

**Why it matters:** In complex contexts, sense-making frameworks that handle emergence work best.

### Common Mistakes

❌ **Using stable-context frameworks in chaos** (SWOT in crisis)
❌ **Over-analyzing in uncertain contexts** (analysis paralysis)
❌ **Treating complexity as complication** (linear thinking for emergent systems)
❌ **Ignoring context changes** (applying yesterday's framework to today's problem)

## Dimension 3: Time Horizon

### What to Explore

How immediate or far-reaching the decision/problem is.

**Key questions:**
- When do you need results?
- What timeframe are you optimizing for?
- Is this urgent or important?
- How will this play out over time?

### Time Horizon Categories

#### Immediate (Days/Weeks)
**Characteristics:**
- Urgent action needed
- Quick decisions required
- Tactical focus
- Short feedback loops

**Best frameworks:**
- OODA Loop (observe-orient-decide-act)
- Eisenhower Matrix (urgent/important)
- Pareto Principle (quick wins)
- Six Thinking Hats (rapid perspective-taking)
- Pre-Mortem (quick risk assessment)

**Why it matters:** Immediate timeframes need fast-cycle frameworks, not deep analysis.

#### Short-term (Months)
**Characteristics:**
- Project-level thinking
- Iteration possible
- Building and testing
- Measurable outcomes

**Best frameworks:**
- Design Thinking (innovation projects)
- Lean Startup (MVP development)
- OKR Framework (quarterly objectives)
- RICE Framework (prioritization)
- Jobs-to-be-Done (product development)
- Theory of Constraints (process improvement)

**Why it matters:** Short-term timeframes need action-oriented frameworks with feedback loops.

#### Long-term (Years)
**Characteristics:**
- Strategic choices
- Hard to reverse
- Compound effects
- Legacy considerations

**Best frameworks:**
- Regret Minimization (life decisions)
- Wardley Mapping (strategic evolution)
- Scenario Planning (future scenarios)
- Second-Order Thinking (long-term consequences)
- Mental Models (timeless principles)
- Business Model Canvas (business design)
- Porter's Five Forces (industry dynamics)

**Why it matters:** Long-term timeframes need frameworks that consider lasting implications.

### Common Mistakes

❌ **Using long-term frameworks for urgent decisions** (Scenario Planning when you need to decide today)
❌ **Using short-term frameworks for strategic choices** (RICE for life decisions)
❌ **Not matching urgency to framework cycle time**
❌ **Confusing urgent with important** (tactical frameworks for strategic problems)

## Dimension 4: Stakeholders

### What to Explore

Who's involved, affected, and needs to buy in.

**Key questions:**
- Who makes the decision?
- Who's affected?
- Who needs to implement?
- How many perspectives matter?

### Stakeholder Considerations

#### Individual Decisions
**Characteristics:**
- Personal choice
- Self-directed
- Own consequences
- Limited coordination

**Best frameworks:**
- Regret Minimization (personal decisions)
- Circle of Competence (knowing limits)
- Second-Order Thinking (personal consequences)
- First Principles (independent thinking)
- Inversion (personal risk management)

**Why it matters:** Individual frameworks focus on personal clarity, not consensus.

#### Team/Department Level
**Characteristics:**
- Group decisions
- Multiple perspectives
- Need coordination
- Shared outcomes

**Best frameworks:**
- Six Thinking Hats (structured perspectives)
- OKR Framework (team alignment)
- Design Thinking (collaborative innovation)
- SWOT Analysis (group analysis)
- Pre-Mortem (team risk assessment)

**Why it matters:** Team frameworks need to handle multiple perspectives constructively.

#### Organizational Level
**Characteristics:**
- Many stakeholders
- Political considerations
- Change management
- System-wide impact

**Best frameworks:**
- McKinsey 7S (organizational alignment)
- Wardley Mapping (strategic communication)
- Scenario Planning (shared understanding)
- Systems Thinking (organizational dynamics)
- Theory of Constraints (systemic bottlenecks)

**Why it matters:** Organizational frameworks need to build alignment and navigate politics.

### Common Mistakes

❌ **Using organizational frameworks for personal decisions** (McKinsey 7S for career choice)
❌ **Using individual frameworks for team problems** (Regret Minimization for product strategy)
❌ **Ignoring stakeholder buy-in needs** (analytical frameworks when politics matter)
❌ **Not considering implementation complexity**

## Dimension 5: Data Availability

### What to Explore

How much information exists vs assumptions/unknowns.

**Key questions:**
- What data do you have?
- What's validated vs assumed?
- Can you get more data?
- How much uncertainty can you reduce?

### Data Availability Categories

#### Rich Data Available
**Characteristics:**
- Historical data
- Market research
- Validated information
- Quantitative analysis possible

**Best frameworks:**
- Porter's Five Forces (industry analysis)
- BCG Matrix (portfolio data)
- Pareto Principle (known patterns)
- SWOT Analysis (known factors)
- Risk Management Frameworks (quantifiable)
- RICE Framework (data-driven prioritization)

**Why it matters:** When data exists, analytical frameworks that leverage it work best.

#### Moderate Data (Mix of Known/Unknown)
**Characteristics:**
- Some validation
- Many assumptions
- Directional data
- Qualitative insights

**Best frameworks:**
- Business Model Canvas (hypothesis-based)
- Scenario Planning (multiple possibilities)
- OODA Loop (continuous learning)
- Mental Models (reasoning frameworks)
- Cynefin Framework (sense-making)

**Why it matters:** Mixed-data contexts need frameworks that work with assumptions explicitly.

#### Mostly Unknowns
**Characteristics:**
- High uncertainty
- Few facts
- Many hypotheses
- Experimentation needed

**Best frameworks:**
- Lean Startup (validated learning)
- Design Thinking (discovery-focused)
- First Principles (reasoning from fundamentals)
- Inversion (avoiding known failures)
- Pre-Mortem (imagining unknowns)
- Jobs-to-be-Done (customer discovery)

**Why it matters:** When data is scarce, frameworks that embrace learning and experimentation work best.

### Common Mistakes

❌ **Using data-heavy frameworks without data** (Porter's Five Forces with no industry data)
❌ **Over-analyzing with insufficient data** (analysis paralysis on assumptions)
❌ **Not distinguishing facts from assumptions**
❌ **Treating hypotheses as validated** (premature confidence)

## Dimension 6: Implementation Complexity

### What to Explore

What's required to execute and who needs to be involved.

**Key questions:**
- Can you implement this alone?
- Does this require team/organizational change?
- How long will implementation take?
- What resources are needed?

### Complexity Categories

#### Simple Implementation (Individual Action)
**Characteristics:**
- Can execute alone
- Clear next steps
- Limited dependencies
- Fast implementation

**Best frameworks:**
- Eisenhower Matrix (personal prioritization)
- Pareto Principle (personal optimization)
- Regret Minimization (personal decisions)
- Circle of Competence (personal boundaries)
- Second-Order Thinking (personal reasoning)

**Why it matters:** Simple implementations need clarity frameworks, not change management.

#### Moderate Complexity (Team Effort)
**Characteristics:**
- Requires team coordination
- Some organizational support
- Multi-week implementation
- Moderate resources

**Best frameworks:**
- OKR Framework (team alignment)
- Design Thinking (team innovation)
- Lean Startup (team experimentation)
- RICE Framework (team prioritization)
- Theory of Constraints (team optimization)
- Six Thinking Hats (team decision-making)

**Why it matters:** Team implementations need coordination frameworks.

#### High Complexity (Organizational Change)
**Characteristics:**
- Organization-wide impact
- Change management required
- Multi-month/year implementation
- Significant resources

**Best frameworks:**
- McKinsey 7S (organizational transformation)
- Wardley Mapping (strategic shifts)
- Scenario Planning (strategic planning)
- Systems Thinking (organizational systems)
- Porter's Five Forces (strategic repositioning)

**Why it matters:** Complex implementations need frameworks that address organizational dynamics.

### Common Mistakes

❌ **Choosing complex frameworks for simple problems** (McKinsey 7S for personal productivity)
❌ **Choosing simple frameworks for complex problems** (Eisenhower Matrix for organizational transformation)
❌ **Ignoring implementation feasibility** (perfect framework, impossible execution)
❌ **Not considering timeline constraints** (long-cycle framework for urgent need)

## Matching Process

### Step 1: Assess All 6 Dimensions
- Problem Type: Strategic / Operational / Innovation / Decision
- Situation Context: Stable / Changing / Uncertain / Complex
- Time Horizon: Immediate / Short-term / Long-term
- Stakeholders: Individual / Team / Organization
- Data: Rich / Moderate / Scarce
- Complexity: Simple / Moderate / High

### Step 2: Find Framework Overlap
Look for frameworks that match multiple dimensions:

**Example:**
- Problem: Innovation
- Context: Uncertain
- Time: Short-term
- Stakeholders: Team
- Data: Scarce
- Complexity: Moderate

**Match:** Design Thinking or Lean Startup (both fit all dimensions)

### Step 3: Consider Trade-offs
When multiple frameworks fit:
- Which dimension is MOST important?
- Which framework matches that dimension best?
- What are the trade-offs?

### Step 4: Explain Why It Fits
Always connect recommendation to user's specific situation:
- "Design Thinking fits because you're [innovating], in [uncertain] environment, with [team], and need to [discover]"
- Not: "Design Thinking is popular"

## Remember

1. **Match to situation, not popularity** - Famous frameworks aren't always right
2. **Explain the why** - User needs to understand fit reasoning
3. **Consider trade-offs** - No perfect framework, acknowledge limitations
4. **Warn about mismatches** - Better to flag issues early
5. **Adapt to context** - Same framework can be applied differently based on dimensions
